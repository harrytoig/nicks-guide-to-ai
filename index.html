Nick's Guide to AI

Learn AI by doing it. A path from zero to AI Native, for anyone aiming at AIOM, Forward Deployed, or any applied-AI role.

by Harry Toig · AI Outcomes Manager at Glean
←→ or ↑↓ to move between sections

No prerequisites. No CS degree required. Four years ago I was in finance at Snowflake. Then I watched some YouTube videos, talked to a guy named Max Comolli, and started building. I am now an AI Outcomes Manager at Glean. The ladder is climbable.

The thesis: learn AI by doing it. Not by reading about it. Not by grinding courses. Not by polishing your resume. Pay $20 a month for Claude, build five things, and be dangerous in six months.

This guide targets a specific end state: AIOM (AI Outcomes Manager) or any adjacent applied-AI role: Forward Deployed Engineer, Solutions Engineer, AI Product Manager, Customer Engineer. These roles sit between the people building frontier models and the customers using them. They're the most gettable AI jobs for non-technical backgrounds.

The North Star: AI Native

Glean's internal framework for AI fluency has three tiers. The hiring bar for AIOM is the top one.

AI Literate

Uses ChatGPT competently. Writes decent prompts. Below the floor.

AI Proficient

Has built agents. Uses Claude Code. Has workflows that measurably change their output.

AI Native ←

Builds habitually. Architects workflows from scratch. Iterates prompts as craft. Debugs failure modes. Can't imagine working without these tools.

AI Native is not a credential. It is what you are after you have shipped enough things. You get there by building.

What You Need to Know

If you read one section in this guide, read this. The specific skills the AIOM path requires. The chapters that follow are the curriculum.

Technical

  • Prompting as craft. Role → Task → Context → Reasoning → Output → Stop Condition. Iteration, not one-shot.
  • Agent design. Retrieval, grounding, guardrails, tool calling, multi-step flows.
  • RAG fundamentals. Explain retrieval-augmented generation to a non-technical Sales Director in 90 seconds.
  • LLM fundamentals. Context windows, reasoning vs. fast mode, evals, why models hallucinate.
  • MCP, Custom Actions, APIs. When to use each. Webhooks, REST, OAuth basics.
  • JSON literacy. Read and edit configs without flinching.
  • Reading code. Not writing Python from scratch. Reading it when an agent you're debugging spits out a stack trace.

Non-technical

  • Customer discovery. Turn ambiguous pain into a shippable agent spec.
  • Change management. Champion frameworks, departmental rollouts, adoption tracking.
  • Writing. Briefings, runbooks, how-to guides customers actually use.
  • Enablement delivery. Train 50-200 people without putting them to sleep.
  • Executive presence. You'll brief CHROs, CTOs, and CEOs on their own AI strategy.

Mindset

Hungry, curious, consultative. You build things for yourself, not for the interview. Low ego, high agency. Resilience with ambiguity. Candidates who drop off when asked to "build or present something" self-select below the bar.

The Three Moves That Matter

In order. Not negotiable.

01

Understand it.

Spend ten hours with the right people explaining what an LLM actually is. Not hype videos. The real ones.

02

Use it every day.

Until you have gut feel for what these things can and can't do, no amount of theory will save you.

03

Build something.

Not a to-do app. Something that only exists because AI exists. The single highest-leverage thing you can do. Everything else is commentary.

If you are short on time, start with #3. You can reverse-engineer the other two from there.

Your First Hour

If you do nothing else this week, do these five things today.

  1. Pay for Claude. $20 per month. Use it for everything: writing, summarizing, planning, thinking. You cannot learn AI on the free tier.
  2. Install Claude Code. npm install -g @anthropic-ai/claude-code. Point it at your laptop. Ask it to do something useful.
  3. Watch Karpathy's "Intro to Large Language Models" (one hour, free). The highest-density AI explainer that exists.
  4. Subscribe to Simon Willison's blog. He reads the internet so you don't have to.
  5. Pick a project from Build. Start today. Finish it this week. Ship it publicly.
Chapter 01

Fundamentals

How LLMs work, what to learn first, and what to skip.

You don't need to understand attention math. You do need a working mental model: what an LLM is, what it can and can't do, how prompts, context, tools, and agents fit together. That's what this chapter buys.

Core (watch these, ~5-6 hours)

  1. Andrej Karpathy, Intro to Large Language Models (1 hr). The single best "what is an LLM" video ever made. Non-technical but substantive. Start here.
  2. Andrej Karpathy, Software Is Changing (Again) (40 min). The Software 1.0 → 2.0 → 3.0 framing. Essential mental model for where this is going.
  3. Andrej Karpathy, "Deep Dive into LLMs like ChatGPT" (3.5 hr, Feb 2025). The full stack: pretraining, fine-tuning, RLHF, why hallucinations happen. Find it on his YouTube channel.
  4. Boris Cherny on Lenny's Podcast (~90 min). Head of Claude Code on product philosophy. Watch this before you try to use Claude Code seriously. Also find his Latent Space appearance for the technical version.

Do (hands-on, ~5 hours)

Tutorials you actually run.

  1. Anthropic's Prompt Engineering Interactive Tutorial (free, 9 chapters of Jupyter notebooks, ~4 hr). The best free prompt engineering course that exists. Do every exercise. This is the single highest-ROI thing on this page.
  2. Code w/ Claude: Prompting 101 (Anthropic applied team, 45 min). Real production prompting patterns, not toy examples.
  3. Read the Claude docs. Start with Agents and Tool Use and Model Context Protocol. Skim them now, come back when you're stuck.

Optional depth

Not required. Worth it if curious.

  • Karpathy, Let's build GPT from scratch (2 hr). Codes GPT live. The highlight of his Zero to Hero series.
  • Karpathy, Neural Networks: Zero to Hero (7 videos, ~15 hr). Builds neural nets from scratch: micrograd → makemore → nanoGPT. If you want internals, the best playlist ever made.
  • 3Blue1Brown, But what is a GPT? and Attention in transformers (~50 min together). Animated visual explainers. For the "I'm not a math person but I want intuition" case.
  • Simon Willison's talks. Running commentary on LLMs as a working tool.

Skip

Be ruthless. Wrong-era resources cost more than they teach.

  • LangChain courses. AIOMs work on agent platforms, not LangChain.
  • fast.ai Practical Deep Learning. Great course, wrong audience. It's for ML researchers, not AIOMs.
  • Hugging Face LLM Course. Fine-tuning and deployment aren't AIOM-core.
  • Most Lex Fridman AI episodes. Occasional gems, mostly length.
  • Anything from pre-2023 on practical LLMs. The field has moved twice since. Theory ages well. Practical takes expire in 18 months.

Four-week path. Week 1: Intro to LLMs + Software Is Changing + Prompt Eng Tutorial (first 3 chapters). Week 2: finish the tutorial + Code w/ Claude: Prompting 101 + read Agents/Tool Use docs. Week 3: Deep Dive into LLMs + Boris Cherny on Lenny's. Week 4: start building.

You don't need to finish every video before you build. You learn faster by building halfway through.

Chapter 02 · The most important chapter

Build

Nine real projects, sorted so each one teaches something the next depends on.

Theory is table stakes. Podcasts are background radiation. Projects are the thing.

If you do nothing else, do this ladder. By rung 7 you can hold your own in any applied-AI interview.

The Rule. Ship every project publicly. GitHub repo + 90-second Loom + LinkedIn post. The artifacts ARE your portfolio. A resume with "interested in AI" is worth nothing. A GitHub profile with nine shipped AI projects is worth a job.

1

Automate one annoying task with Claude Code

Difficulty 1 · 2-4 hours

Pick one friction point you actually hate: inbox triage, meal planning from fridge photos, weekly calendar summary, workout logging. Install Claude Code, describe the task, iterate until it works.

Teaches: how agents behave, why context matters, prompts as specifications. Easy entry and a preview of how you'll spend days as an AIOM.

2

Build a chatbot with the Claude API

Difficulty 1 · one weekend

Single-file Python script hitting the Claude API with a system prompt (a persona). Use the Anthropic quickstart.

Teaches: API calls, tokens, system vs. user messages. Removes the abstraction so you see what's actually happening behind every agent builder.

3

RAG over your own notes

Difficulty 2 · one weekend

Start easy: drag your journal, Notion export, or Obsidian vault into Claude Projects and chat with it. Then rebuild from scratch with LlamaIndex or the Anthropic RAG cookbook.

Teaches: embeddings, chunking, and the truth that retrieval quality is the whole ballgame. You need to explain RAG to a non-technical stakeholder in 90 seconds. Build one, and you can.

4

Build a tool-use agent

Difficulty 2 · one week

Claude with the tool_use API. A bot that reads your calendar, checks weather, and texts you a day plan. Or a finance agent that reads Plaid transactions and yells at you about your grocery bill.

Teaches: function calling, agent loops, and (most importantly) why agents fail. Ambiguous tools, bad state handling, retry strategies, when to ask a human. The meat of the AIOM job.

5

Install MCP servers and write a custom one

Difficulty 2 · a few hours

Install two or three MCP servers in Claude Desktop: Gmail, Google Calendar, Plaid. Then write a tiny custom MCP server using Anthropic's Python template. Point Claude at something weird in your life.

Teaches: what MCP is and why it's the direction the industry is going. Table stakes now; a floor in six months.

6

Build an agent on a no-code platform

Difficulty 1 · 2-4 hours

Glean Agent Builder if you're at a Glean customer. Otherwise Dify, Flowise, LangFlow, or n8n's AI nodes. Build a three-step agent with retrieval, an LLM step, and a tool call.

Teaches: what "building an agent" looks like when the platform abstracts the API. Exactly what AIOMs do with customers all day.

7

Write a simple eval for one of your agents

Difficulty 3 · half a day

Pick an agent you already built. Write 10-20 test cases with expected outputs. Run them. Score pass/fail. Change one thing in the prompt. Score again.

Teaches: you cannot ship what you cannot measure. The skill that separates hobbyists from people who get hired. See Anthropic's evaluation framework guide.

8

Deploy a public chatbot with the Vercel AI SDK

Difficulty 2 · one weekend

Clone vercel/chatbot, customize, deploy to a public URL. Get 10 friends to use it. Watch the logs.

Teaches: streaming, React hooks, and (crucially) shipping to real users. The gap most tutorials skip.

9

Make Claude Code your daily driver

Difficulty 3 · ongoing

Custom slash commands. Rules files. Skills. MCPs. Replace five manual workflows with Claude-Code-driven ones. Commit your setup to a repo.

Teaches: all of it. The project AIOM interviewers want to see. See Claude Code for the setup.

Optional depth

For the curious, not AIOM-core.

  • Karpathy's makemore + nanoGPT (Difficulty 4, 1-2 weeks). Watch Zero to Hero, code along, train a tiny model on your own writing. Credibility-bank.
  • Fine-tune a small model (Difficulty 3, one week). Gemma 3 or Phi-3 on Colab free tier. Teaches data quality beats model size.
  • Voice app: Whisper + Claude + ElevenLabs (Difficulty 3, one week). Record, transcribe, reason, speak. Teaches multi-model pipelines.

Ideas if you want more

  • Home-finance agent that categorizes Plaid transactions and flags weird spending.
  • "What did I do this week" agent that summarizes your git commits, calendar, and journal.
  • Personal health tracker that ingests Oura data, lab results, and meal photos.
  • Slack bot that reads your team's channels and drafts a weekly digest.
  • Agent that watches your Gong or Granola transcripts and drafts follow-up emails.

One rule before you start. Pick the smallest version you can finish this week. Shipped and ugly beats unshipped and beautiful. You learn 10x more from one deployed project than from three half-built ones.

After you ship. GitHub README → 90-second Loom → LinkedIn post → tweet it. Simon Willison, Ethan Mollick, and Alex Albert actually reply to people shipping good work. Ship and they'll find you.

Chapter 03

Claude Code

The daily-driver setup you should steal.

Claude Code is Anthropic's CLI. Type plain English; it writes code, edits files, runs commands, and hands back results. Better than ChatGPT for anything involving your actual filesystem: docs, code, automations, new projects from zero.

Single biggest productivity unlock I've found in two years. Pay the $20 a month for Claude and use the hell out of it.

Install

npm install -g @anthropic-ai/claude-code

You need Node.js. If you don't have it, brew install node. If you don't have Homebrew, install it first.

Then cd to a folder and type claude. That is it.

The six things worth configuring

Ignore all of this for your first week. Just start using Claude Code on real tasks. When it gets frustrating in the same way twice, come back here.

1. CLAUDE.md

A CLAUDE.md file at the root of any project gets read every session. Use it to tell Claude who you are, how to talk to you, what tools to prefer, and what to avoid. Put one at ~/CLAUDE.md so Claude knows you across every project.

2. Rules files

For longer preferences, split into ~/.claude/rules/*.md, one file per topic. Mine: personal-context, tool-routing, parallel-execution, memory-and-persistence, reminders. They auto-load every session.

3. Slash commands

A slash command is a reusable prompt you invoke with /name. Drop a markdown file into ~/.claude/commands/. Good ones to build: /morning, /evening, /standup, /brief [customer], /log. If you use the same prompt three times, make it a command.

4. Skills

Skills are commands' more structured cousin. Multi-step, can spawn sub-agents, can ship with their own domain knowledge. If a command gets longer than 30 lines or needs sub-steps, promote it to a skill.

5. MCPs (Model Context Protocols)

Plug-in tool servers that let Claude talk to external services. Install early: Google Calendar / Gmail MCP, Granola MCP for meetings, Plaid MCP for finance, Glean MCP for work knowledge, Computer Use MCP for desktop control. This is where the rabbit hole begins.

6. Memory

Claude Code writes auto-memory notes to ~/.claude/projects/[project]/memory/ across sessions. It remembers preferences, corrections, tool mistakes. You can prompt directly: "remember that I prefer X over Y." Pattern: MEMORY.md as an index under 200 lines, topic files for tool-lessons, preferences, debugging.

First week

  • Ask Claude to read your entire Downloads folder and categorize what's in it.
  • Summarize your last 30 days of journal entries. Find out what you're obsessing over.
  • Build a one-page website from a rough description and deploy to Vercel.
  • Write a slash command for your morning routine: calendar check, top news, day plan.
  • Point it at an annoying manual process you do weekly and have it write the bash script.

Use Claude Code as your primary interface for a week. You won't want to go back.

Warning. Claude Code can do destructive things. Don't give it unsupervised access to production databases. Don't let it rm -rf blindly. Always confirm before a git push.

Chapter 05

The Role

What AIOMs actually do, and what skills you need to get there.

You're not going to become an AI researcher at Anthropic without a PhD. Fine. The most valuable roles in AI right now are not research roles. They're the people between a research lab and a paying customer: Forward Deployed Engineers, Applied AI Engineers, Customer Engineers, AI Outcomes Managers. They build prototypes against real customer data, run enablement, turn ambiguous pain into shippable agents, and feed what they learn back into the product.

A typical AIOM week

A real week. Not polished. Representative.

Monday
Build a campaign-brief-generator agent for a customer's marketing team that pulls from Snowflake. 90 minutes use-case discovery, 90 prompt iteration, 60 testing.
Tuesday
Roll out an agent champion program at a 3,000-person enterprise. Train 15 department leads. Write a best-practices doc.
Wednesday
Help another AIOM unblock their customer. Post 500 words in #aiom-team on a technical pattern. Interview a candidate.
Thursday
Executive briefing at a Fortune 500. Present agent portfolio, usage numbers, 90-day plan. Schedule the next six weeks of work.
Friday
Ship a new Claude Code skill to automate part of your own job. Run a 160-person webinar on prompt engineering. Log everything.

You are a hybrid consultant / prompt engineer / product thinker. Scored on adoption (is the customer actually using it?) and power agents (are the agents still used weekly, months later?). Both metrics reward learn-by-doing. Neither rewards talk.

How you get there

The whole guide is the answer. A summary:

  1. Build in public. Every project: GitHub repo, 90-second Loom, LinkedIn post. Your visible work becomes the resume.
  2. Cold outreach is a working demo. Ship something on the target company's API. 90-second walkthrough. Send it with one specific question. Reply rate is 10x generic outreach.
  3. Pick a vertical and go deep. Domain expertise plus AI fluency is rarer than either alone.
  4. Write. A Substack with five posts of "here is what I built and what I learned" outperforms any polished resume.
  5. Skip the bootcamps. Reforge or Refonte only buys you a cohort you can replicate with a Discord server.
  6. Treat the job search itself as a project. Build an agent that finds roles, drafts tailored cover letters, tracks applications. Show it to your interviewers.
The single piece of meta-advice

The gap between one shipped project and zero is wider than any credential in the industry right now.

Closing that gap is a weekend of work. So go.
